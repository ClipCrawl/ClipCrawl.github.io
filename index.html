AMBER

Measuring the ambiguity penalty in corpus-level video moment retrieval. A benchmark that preserves the whole match set — every match edge verified on pixels — plus the agentic verification method built on top of it, and the honest negative results.

1,106 queries 3 video corpora every edge pixel-verified penalty +19.9 @1 on ambiguous queries

The problem

Retrieval scores lie when the query is ambiguous.

35distinct videos in our corpus genuinely contain a moment matching one real query — “a person enters a room through a door.” One is marked gold; 34 correct answers get penalized.

Every widely-used corpus-level VCMR benchmark marks exactly one gold moment per query. But queries are rarely that specific. A system that returns any of the non-annotated true matches is scored wrong for being right — so reported accuracy is a lower bound of unknown tightness. AMBER makes that looseness measurable.

The dataset

1,106 queries, and every match they have.

Built from three public corpora into one ambiguity graph per query, then projected into two evaluation tracks. Construction runs entirely on open-weight local models.

Construction — caption-driven, pixel-gated

1 · SEGMENT + CAPTION

Describe events

Content-adaptive event segmentation; one local-VLM caption per event. Captions are the sole semantic medium.

2 · GENERATE

Query from caption

An LLM writes coarse + fine queries — never reusing 3 consecutive caption words, so matching stays semantic.

3 · MERGE

Discover ambiguity

Near-identical queries from different videos merge — two events spawning the same query are an ambiguity edge.

4 · CORPUS SCAN

Entailment sweep

Each query is checked against every caption in the corpus (BM25 → LLM entailment, k=200) → the full text-space match set.

5 · PIXEL GATE

Verify every edge

A VLM re-checks every edge on real frames — including the anchor. A failed anchor drops the whole query (gate-all).

6 · RELEASE

Dual-track record

Video-disjoint splits, checksummed manifest, metric unit tests, per-arm predictions.

The two tracks

U — query-unique

Honest single-gold

558 queries whose verified match set is a single moment. Standard R@K / VCMR@IoU are exact here — there is no un-annotated true match to wrongly penalize.

C — recall-complete

The full match set

All 1,106 queries with every verified match published. Multi-positive metrics (Correct@K, mAP@R, NDCG@IoU): a system is right if it finds any true moment.

Live examples — real records, real frames

Label quality, stated plainly. Captions, queries, entailment and the pixel gate share a model family, so we audit from outside it: an independent-family caption audit (1.00 / 0.91 / 0.94), a cross-family re-gate (InternVL3 confirms 89.8% of shipped edges; scan-mined edges are the weakest stratum at 82–85%), and a measured construction-bias probe — BM25 over the construction captions reaches 53–80% C-track@1, far above every pixel system, so the captions are withheld from the evaluation release. A human edge audit is the remaining step and is planned.
The measurement

One ranking, scored two ways.

The gap between them is the ambiguity penalty — the rate at which single-gold scoring marks a correct retrieval wrong.

single-GT (prior benchmarks)

Only the anchor ★ counts.
Top-1 = m₂ ⇒ scored wrong

hit@1 = 0
C-track (AMBER)

Any m ∈ M counts.
Top-1 = m₂ ⇒ correct

hit@1 = 1
ambiguity penalty
Δ@k = C-track@k − single-GT@k

= the rate of being wrong for being right.
Built-in control: |Mq| = 1 ⇒ Δ ≡ 0 — measured exactly 0.000 for every system, at every k, in every source.

True rank-1 accuracy on ambiguous queries is 2.3× the single-gold score. Per source: Charades +13.3 · DiDeMo +23.8 · QVH +26.0. Test-split-only, with bootstrap CI: +15.2 [8.7, 22.8].

External validity — the penalty exists for human queries too. Running the original human-written DiDeMo annotations through the same verification machinery yields a multi-video penalty of +23.9 — matching the generated-query value (+23.8) almost exactly; human queries are even more ambiguous than generated ones (0.56 vs 0.41). QVHighlights: +50.0 (n=16). The mis-scoring is a property of the source benchmarks' annotations, not of our query generator.

Single-gold scores barely move while honest scores climb: the better the system, the more often its top-1 is a true-but-unannotated match — single-gold leaderboards compress exactly the differences they exist to measure (gap vs CLIP: +9.1, CI [1.1, 18.2]). A corpus-moment-level analogue of pooling bias in IR evaluation.

The method

An agentic loop that pays for pixels only where they matter.

A VLM policy over a cost-metered tool registry, with an episodic memory shared across queries — because AMBER's match sets expose exactly which queries share moments.

Negative results, reported as such

SFT distillation: null. An apparent +2.7 gain under a naive anchor-disjoint split was entirely supervision leakage (39% of test queries had a gold video the student trained on). After decontamination: −0.7 vs zero-shot (p=0.83). We ship the decontaminated numbers and a per-record supervision_safe flag.
Multi-positive RL reward vs anchor-only: no significant difference at our training scale (357 clean queries, shortlist-depth-5 regime where the two rewards coincide on 89% of queries). A redesigned run targeting the regime where anchor-reward starves is in progress.
Granularity dependence: not supported. Agent gains on fine vs coarse queries (+14.4 vs +11.2) are within noise (z=0.52). The agent helps significantly on both.

Limitations

Match sets are complete only relative to the released pools (199–933 videos/source); a human edge audit and the Charades-STA natural-query control are planned; the frontier-agent arm uses a proprietary assistant (prompt and all 259 trajectories released); MOMA was excluded after failing the caption-faithfulness gate at every captioner setting tried (0.81–0.87) — a documented VLM ceiling on sports on-screen text.
